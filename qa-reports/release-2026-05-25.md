## 1. CI status — WAS MISSING → NOW PROPOSED

Before this cycle, `/Users/skypie/Games/pacman-code-trainer/.github/workflows/` did **not exist**. Pac-Man had zero automated checks despite being a live, public site. Even a typo in `cards.js` could ship straight to production.

**This cycle adds** (on `release/auto-2026-05-25`, NOT merged):

`.github/workflows/ci.yml` — three checks per push/PR:
1. `node --check cards.js` — parses the deck file as JS. Catches syntax errors before deploy.
2. `node test/validate-cards.js` — runs the zero-dep data validator. No-op on main today; auto-activates when `test/pacman-cards-data-validator-2026-05-25` merges.
3. `grep` smoke-check that `index.html` still references `<script src="cards.js">`. Catches accidental script-tag removals.

All three are zero-dependency — no `npm install`, no Node packages. Workflow finishes in well under a minute.

**Not added (out of scope):**
- No Playwright/headless-browser test of the actual flashcard UI. Could be added later if Gary wants browser-level integration tests.
- No Lighthouse/Pa11y a11y check on every PR. Could be Alex's call.
